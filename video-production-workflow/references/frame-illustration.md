# 绘图阶段

## 使用时机

用户已有分镜，需要帧图提示词、缩略图提示词、图像生成方向或统一视觉规范时使用本文件。

开始前读取 `style-guide.md`。如果用户指定画风，或需要为全片选择画风，再读取 `visual-styles.md` 做路由，然后只读取选中的 `styles/*.md` 风格文件。

不要改动旁白，除非用户明确要求。

## 工作流

### 步骤 1：生成提示词

1. 读取已批准的分镜。
2. 为全批次选择一个主制图风格。若用户未指定，优先选择最适合内容的风格，并在输出中写明选择理由。可选风格见 `visual-styles.md`，具体规则见对应 `styles/*.md`。
3. 为每个 shot 生成一条图像提示词。
4. 保持一致：线条粗细、人物比例、颜色、文字处理、留白方式必须属于同一视觉系统。
5. 默认将 `Shot 01`、`SHOT_01`、`shot-01`、`镜头01` 等 shot 名称或编号列入负面约束，禁止它们出现在画面内。只有用户明确要求展示 shot 标签时才允许省略此约束。
6. 将提示词写入 `04-image-prompts.md`。

### 步骤 2：实际生成图片

提示词生成后，**必须实际调用制图模型为每个 shot 生成图片**，不要只交付提示词就结束。

- 使用当前环境可用的制图模型。图片必须由模型直接生成；禁止改用 SVG、Canvas、Pillow、本地脚本绘图或网页截图伪造模型成图。没有可用制图模型时，停止绘图阶段并明确说明缺失条件。
- **生成图片时必须传入与画面比例匹配的实际分辨率**：
  - 16:9 横屏 → `1920x1080`（或工具支持的最接近 16:9 的分辨率）
  - 9:16 竖屏 → `1080x1920`
  - 1:1 方形 → `1080x1080`
  - 用户指定其他比例时，按比例计算对应分辨率。
- 按 shot 编号顺序生成。保留制图模型直接返回的正常栅格图片格式，支持 `.png`、`.jpg`、`.jpeg`、`.webp`；保存为 `images/<shot-id>.<ext>`（如 `images/shot-01.webp`）。同一项目优先统一格式，但不得为了统一扩展名而使用本地绘图或截图转换替代制图模型。
- 生成时使用步骤 1 中的完整提示词（含风格、构图、主体、动作、颜色、文字、负面约束和一致性说明）。
- 所有 shot 的图片必须使用相同分辨率，确保渲染阶段无需缩放。
- 如有条件，可并行生成多个 shot 的图片以加速。
- 每张图片生成后，检查图片是否成功保存、是否符合提示词描述和指定分辨率；失败的 shot 立即重试或调整提示词后重试。
- 缩略图方向同样需要实际生成 3 张缩略图图片，保存为 `images/thumbnail-01.<ext>`、`images/thumbnail-02.<ext>`、`images/thumbnail-03.<ext>`。缩略图分辨率与正文帧图一致。

## 风格锁定

每个项目只选一个主风格。不要在同一批分镜里混用小黑、黑底象征、扁平图标、漫画大字等风格，除非用户明确要求做风格探索。

所有风格通用要求：

- 默认 16:9，除非用户指定其他比例。
- 一帧一个核心视觉概念。
- 文字少而短，画面文字使用用户指定语言；未指定时使用用户输入的主要语言。
- shot 标签只用于提示词章节标题、文件命名和生产管理，默认不得成为画面文字。除非用户明确要求，图片中不得出现任何 shot 名称、镜头编号或类似生产标签。
- 画面元素服务旁白，不要为了装饰增加信息。
- 提示词必须包含风格名、构图、主体、动作、颜色、文字、负面约束和一致性说明。

## 输出格式

```markdown
### Shot 01
Narration: ...
Style: ...
Prompt: ...
Negative prompt: ...
Consistency note: ...
```

## 缩略图方向

为主缩略图提供 3 个方向：

1. `问题 + 人物`
2. `单一符号 + 大留白`
3. `左右反差`

缩略图优先无字或极短文字。画面文字使用用户指定语言；未指定时使用用户输入的主要语言。

缩略图方向也需要实际生成 3 张图片，保存为 `images/thumbnail-01.<ext>`、`images/thumbnail-02.<ext>`、`images/thumbnail-03.<ext>`。

## 验收

提示词集合必须像同一视觉系统，不像多个风格拼贴。若选择小黑怪诞正文配图风格，额外按 `styles/xiaohei-absurd-article.md` 中的 QA 检查。

绘图阶段的最终验收标准：

1. `04-image-prompts.md` 已写入，包含所有 shot 的提示词。
2. `images/` 目录下每个 shot 都有对应的模型原始栅格图片文件，格式为 `.png`、`.jpg`、`.jpeg` 或 `.webp`。
3. `images/` 目录下有 3 张缩略图图片。
4. 所有图片视觉风格一致，不存在混用风格的情况。
5. 失败的图片已重试或修复。
6. 除非用户明确要求，所有成图均不含 shot 名称、镜头编号或类似生产标签；若出现则修改提示词并重新生成。
