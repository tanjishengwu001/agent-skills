# 完整工作流

## 使用时机

用户要求"全流程""项目包""从选题到脚本/分镜/提示词""帮我做一期视频生产包"时，或未明确指定单阶段时，使用本文件。

本 skill 默认触发全流程。本文件只描述编排。执行每一步时再加载对应 reference。

## 阶段顺序

### 阶段 1：选题阶段

读取 `topic-selection.md` + `style-guide.md`。

- 只在必要时询问缺失约束：受众、垂类、禁区、平台、是否必须绑定热点或产品。
- 生成 8-12 个选题候选，语言跟随用户需求。
- 对每个候选评分（好奇心、反直觉、证据密度、视觉性、语言传播性）。
- 明确推荐前 2-3 个，说明原因和情绪弧线。
- **选题门禁（不可跳过）**：停在选题阶段，要求用户：
  1. 确认最终选题（选择编号或方向）。
  2. 设置目标视频时长（如 3 分钟、5 分钟、8 分钟）。
- **即使用户说"不要打断""全自动""一口气跑完"等，选题门禁也不可跳过**，必须等待用户主动选择选题并设置时长。
- 不要提前写脚本。

### 阶段 2：脚本阶段

读取 `script-writing.md` + `style-guide.md`。

- 把已确认选题写成目标语言旁白。
- 根据用户设置的目标视频时长，严格按 `script-writing.md` 中的字数预算与校验规则控制脚本长度。
- 保持好奇、证据、反思结尾。

### 阶段 3：分镜阶段

读取 `storyboard-generation.md` + `style-guide.md`。

- 把脚本拆成细粒度 shots。
- 每个 shot 默认 1-2 句旁白。
- 每行分镜包含：Shot 编号、Narration、Visual、Mood/Transition。

### 阶段 4：分镜确认门禁

分镜生成后，**必须停下**，向用户展示并要求确认以下三项：

1. **分镜稿**：镜头拆分是否合理、旁白是否准确、节奏是否舒适。
2. **画面比例**：默认 16:9，询问用户是否需要其他比例（如 9:16 竖屏、1:1 方形）。
3. **制图风格**：根据脚本内容按 `visual-styles.md` 的选择规则给出**推荐风格**，并**列出全部5种风格**供用户选择，或用户自定义风格。

**门禁例外**：如果用户在触发 skill 时明确说了"直接执行全流程""不要打断""全自动""一口气跑完"等指令，则跳过本门禁，按以下默认值自动决策并继续：
- 画面比例：16:9
- 制图风格：根据 `visual-styles.md` 的选择规则自动匹配

确认通过后（或门禁被跳过时），进入绘图阶段。

### 阶段 5：绘图阶段

读取 `frame-illustration.md` + `style-guide.md`。

- 为每个 shot 生成统一风格提示词，写入 `04-image-prompts.md`。
- 如果用户在分镜门禁中指定了风格，读取 `visual-styles.md`，再只读取选中的 `references/styles/*.md` 风格文件。
- 如果用户自定义了风格，直接使用用户描述。
- 生成 3 个缩略图方向，写入 `05-thumbnail-directions.md`。
- **提示词生成后，必须实际调用制图模型为每个 shot 生成图片**。保留模型直接返回的 `.png`、`.jpg`、`.jpeg` 或 `.webp` 栅格格式，保存到 `images/<shot-id>.<ext>`；禁止用 SVG、Canvas、Pillow、本地脚本绘图或网页截图替代制图模型。
- **生成图片时必须传入与画面比例匹配的实际分辨率**（如 16:9 → 1920x1080，9:16 → 1080x1920，1:1 → 1080x1080）。所有 shot 使用相同分辨率。
- 缩略图也需实际生成 3 张图片，保存到 `images/thumbnail-01.<ext>` 至 `images/thumbnail-03.<ext>`，分辨率与正文帧图一致。
- 每张图片生成后检查是否成功保存且尺寸正确；失败的立即重试或调整提示词后重试。

### 阶段 6：配音阶段

读取 `voice-generation.md`。

- 从分镜表的 `Shot` 与 `Narration` 列生成逐 shot 旁白音频。
- 默认使用 Edge provider；保留可切换的 provider 参数与统一 manifest。

### 阶段 7：视频渲染阶段

读取 `video-rendering.md`。

- 使用逐 shot 图片和音频生成 MP4。
- 固定 30fps；每个镜头时长以对应音频时长为准。
- 输出分辨率自动从实际图片检测，无需手动指定。

## 选题门禁规则

选题门禁是**不可跳过**的强制门禁。无论用户以何种方式触发全流程（包括"全自动""不要打断""一口气跑完"等），都必须在选题阶段停下，等待用户：

1. 主动选择/确认最终选题。
2. 设置目标视频时长。

只有当用户明确确认了选题和时长后，才能进入脚本阶段。

如果用户一开始就给出确定题目（例如"就做《为什么你停不下来刷短视频》"），可视为选题已确认，但仍需询问用户目标视频时长。

## 分镜门禁规则

分镜门禁是**可跳过**的确认门禁。默认行为是停下等用户确认分镜稿、画面比例、制图风格三项。

跳过条件：用户在触发 skill 时明确说了"直接执行全流程""不要打断""全自动""一口气跑完"等指令。

跳过时自动决策：
- 画面比例：16:9
- 制图风格：根据 `visual-styles.md` 的选择规则自动匹配

## 交付结构

完整项目包必须保存到当前工作空间目录，并按话题组织目录，避免多个话题互相覆盖。

默认目录结构：

```text
outputs/<topic-slug>/
├── index.md
├── 01-topic.md
├── 02-script.md
├── 03-storyboard.md
├── 04-image-prompts.md
├── 05-thumbnail-directions.md
├── 06-evidence-notes.md
├── audio/
│   ├── voiceover-manifest.json
│   └── <shot-id>.mp3
├── images/
│   ├── <shot-id>.<ext>
│   └── thumbnail-01.<ext> ... thumbnail-03.<ext>
└── video/
    ├── final.mp4
    └── video-manifest.json
```

要求：

- `<topic-slug>` 使用短横线命名，优先从确认后的题目生成；如果同名目录已存在，追加日期或短序号避免冲突。
- `<ext>` 必须是制图模型直接返回的 `.png`、`.jpg`、`.jpeg` 或 `.webp`；正文帧图与缩略图优先使用同一种格式。
